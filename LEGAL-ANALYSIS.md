## What's Been Done So Far — Legal Assessment

### Clean (no risk)
- Extracting questioning *patterns* and *methodology* from lectures
- Building prompt engine based on the method (5-phase framework)
- Transcribing CC-licensed audio for personal R&D use
- Architecture, UI/UX, session memory schema — all original work

### Gray Zone (fine for dev, strip before production)
- Exact Pierre phrasing in prompt templates (`pierre-questioning-patterns.md`)
- Referencing "Pierre Grimes" by name in internal docs
- Using lecture transcripts as training/information source

### Red Line (do not cross without licensing)
- AI representing itself as or embodying Pierre
- Using Pierre's name/likeness in marketing or product UI
- Reproducing lecture content verbatim in the product
- Claiming official endorsement or affiliation with the Noetic Society

---

## Dev Phase Strategy

1. **Fair use for R&D** — Internal prototype development using copyrighted material is strong fair use. Not distributing transcripts, using them to inform original product.

2. **Derive, don't copy** — AI should ask *like* Pierre, not *quote* Pierre. Current prompt engine already follows this discipline.

3. **Name the method, not the man** — Use "Dialectical Dream Analysis" or "Philosophical Dialogue Method" in product. Save Pierre connection for when rights are secured.

4. **Strip exact quotes before external beta** — Any verbatim Pierre language in prompt templates must be paraphrased before any user sees it.
